# Valenty V2 Plan: UI-First Component Testing

## Vision

Valenty's primary workflow guides developers toward **testing through the UI**,
using the `valentyTest` pattern. The AI skills, MCP server, and CLI all push
developers to write tests that exercise the real frontend with stubbed backends
ÔÇö not just test business logic in isolation.

The `@visibleForTesting` factory pattern enables stubbing highly coupled
dependencies with **minimal production code changes** (1 line per dependency).

> Based on [Valentina Jemuovic's Modern Test Pyramid](https://journal.optivem.com)
> and [Frontend Component Tests](https://journal.optivem.com/p/tdd-in-legacy-code-component-tests-frontend)

---

## Two APIs (Both supported)

### Primary: `valentyTest` (UI-first component tests)

```dart
orderTest(
  'should show order total after placing order',
  (system, backend) async {
    backend.stubProduct(sku: 'APPLE1001', price: 2.50);
    backend.stubOrderCreation(totalPrice: 12.50);

    await system.openApp();
    await system.selectProduct('APPLE1001');
    await system.setQuantity(5);
    await system.placeOrder();

    await system.verifyConfirmation('Total: \$12.50');
  },
);
```

**When to use:** Flutter apps ÔÇö the default. Tests real widgets, real navigation,
real user interactions. Backend and external systems are faked.

### Secondary: Typed builders (pure logic tests)

```dart
OrderScenario('should calculate base price')
    .given.product().withUnitPrice(20.00)
    .when.placeOrder().withQuantity(5)
    .then.order().hasBasePrice(100.00)
    .run();
```

**When to use:** Pure Dart packages, complex domain logic without UI, or when
compile-time safety of the builder chain is more valuable than UI coverage.

## Architecture

```
valentyTest / featureTest
    |
    +-- BackendStubDsl           <-- Configures fakes (infrastructure)
    |   |
    |   +-- @visibleForTesting   <-- Minimal production code changes
    |       factory overrides        (1 line per dependency)
    |
    +-- SystemDsl                <-- User-facing domain language
    |   |
    |   +-- UiDriver             <-- Wraps WidgetTester
    |       |
    |       +-- pump, tap,       <-- Real Flutter widget testing
    |           find, enter
    |
    +-- testWidgets              <-- Flutter test framework
```

### Layer responsibilities

| Layer | Responsibility | Example |
|-------|---------------|---------|
| `featureTest()` | Creates driver, stubs, DSL; wraps testWidgets | `authTest('desc', (system, backend) {...})` |
| `BackendStubDsl` | Configure what fakes return | `backend.stubLogin(token: 'abc')` |
| `SystemDsl` | User actions + assertions in domain language | `system.login(email, password)` |
| `UiDriver` | Translates domain actions to widget interactions | `driver.tap(find.byKey('loginBtn'))` |
| `@visibleForTesting` | Production code hooks for dependency injection | `AuthService.httpFactory = () => fake` |

### Phase 1: Core DSL classes in valenty_test

**New files:**


**`valentyTest()`:**
```dart
void valentyTest(
  String description,
  Future<void> Function(SystemDsl system, BackendStubDsl backend) body, {
  required Widget Function() createApp,
  required BackendStubDsl Function() createBackend,
  required SystemDsl Function(WidgetTester tester, BackendStubDsl backend) createSystem,
}) {
  testWidgets(description, (tester) async {
    final backend = createBackend();
    backend.apply();  // override singletons
    try {
      await tester.pumpWidget(createApp());
      final system = createSystem(tester, backend);
      await body(system, backend);
    } finally {
      backend.restore();  // reset singletons
    }
  });
}
```

**Feature-specific wrapper (generated by AI):**
```dart
void orderTest(
  String description,
  Future<void> Function(OrderSystemDsl system, OrderBackendStub backend) body,
) {
  valentyTest(
    description,
    (system, backend) => body(system as OrderSystemDsl, backend as OrderBackendStub),
    createApp: () => const MyApp(),
    createBackend: OrderBackendStub.new,
    createSystem: (tester, backend) => OrderSystemDsl(
      OrderUiDriver(tester),
    ),
  );
}
```

**SystemDsl base class:**
```dart
abstract class SystemDsl {
  SystemDsl(this.driver);
  final UiDriver driver;
}
```

**BackendStubDsl base class:**
```dart
abstract class BackendStubDsl {
  /// Override singleton factories with fakes.
  void apply();

  /// Restore original factories.
  void restore();
}
```

**UiDriver base class (wraps WidgetTester):**
```dart
abstract class UiDriver {
  UiDriver(this.tester);
  final WidgetTester tester;

  Future<void> pumpApp(Widget app) async {
    await tester.pumpWidget(app);
  }

  Future<void> tap(Finder finder) async {
    await tester.tap(finder);
    await tester.pumpAndSettle();
  }

  Future<void> enterText(Finder finder, String text) async {
    await tester.enterText(finder, text);
    await tester.pumpAndSettle();
  }

  Future<void> verifyVisible(Finder finder) async {
    expect(finder, findsOneWidget);
  }

  Future<void> verifyText(String text) async {
    expect(find.text(text), findsOneWidget);
  }
}
```

## @visibleForTesting Stub Patterns

### Common Flutter dependencies

| Dependency | Production code change | Stub pattern |
|------------|----------------------|-------------|
| `Dio()` | `@visibleForTesting static Dio Function() dioFactory = Dio.new` | `dioFactory = () => fakeDio` |
| `SharedPreferences.getInstance()` | `@visibleForTesting static Future<SharedPreferences> Function() prefsFactory` | `prefsFactory = () async => fakePrefs` |
| `FirebaseFirestore.instance` | `@visibleForTesting static FirebaseFirestore Function() firestoreFactory` | `firestoreFactory = () => fakeFirestore` |
| `FirebaseAuth.instance` | `@visibleForTesting static FirebaseAuth Function() authFactory` | `authFactory = () => fakeAuth` |
| `DateTime.now()` | `@visibleForTesting static DateTime Function() clock = DateTime.now` | `clock = () => DateTime(2025, 1, 1)` |
| `GetIt.I<T>()` | Already injectable | Register fakes in GetIt |
| `Riverpod ref.read()` | Use `ProviderContainer` overrides | `container.overrides = [provider.overrideWith(...)]` |

### The rule: 1 line per dependency, zero behavior change

```dart
// BEFORE (legacy code ÔÇö untestable)
class OrderService {
  Future<Order> place(Product p, int qty) async {
    final response = await Dio().post('/api/orders', data: {...});
    return Order.fromJson(response.data);
  }
}

// AFTER (1 line added ÔÇö fully testable)
class OrderService {
  @visibleForTesting
  static Dio Function() dioFactory = Dio.new;  // <-- THIS IS THE ONLY CHANGE

  Future<Order> place(Product p, int qty) async {
    final response = await dioFactory().post('/api/orders', data: {...});
    return Order.fromJson(response.data);
  }
}
```
